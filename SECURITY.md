# Security Policy

## Supported versions

At this stage, only the latest `main` branch is actively supported.

## Reporting a vulnerability

Please do not open public issues for security vulnerabilities.

Instead:

1. Open a private security advisory in GitHub (preferred), or
2. Contact the repository owner directly.

Include:

- Steps to reproduce
- Impact assessment
- Suggested mitigation (if available)

We will acknowledge reports promptly and work on a fix and coordinated disclosure.
